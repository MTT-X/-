行业背景与痛点

航运保险市场

  • 中国航运保险市场超 500亿 RMB
  • 年增速约 8-10%
  • 核心玩家：平安、人保、太平洋
  • 科技赋能是行业趋势

核心痛点

  • 货品描述是自由文字，无统一格式
  • 风险评估依赖人工经验判断
  • 处理效率低，单票耗时 5-15分钟
  • 分类标准不统一，定价偏差大
关键洞察：从"散装化工液体"到"建筑设备零件若干"，2.8万条描述背后是真实的效率损失和定价风险。

竞品与方案调研

方案类型代表优势不足
传统规则引擎各保险公司自建可控、可解释维护成本高、覆盖率有限
ML分类Concirrus, Planck自适应、高精度需标注数据、黑箱决策
LLM方案Zest AI等理解力强、零样本成本高、幻觉风险
我们的选择：规则匹配方案 — 适配业务现状（无标注数据），高可解释性，快速迭代，低成本部署。论证合理比技术复杂度更重要。

错误分析：子串误匹配的发现与修复

关键发现

95.3% 的拒保案例仅命中 1个 关键词，存在大量子串误匹配：
描述误匹配词误判类别
CARDBOARD FILE BOXcar in cardboard车辆
AMUSEMENT MACHINE PARTSart in parts艺术品
POLYURETHANE...SHIPPERship in shipper船舶
修复方案：英文短词(≤3字符)使用正则整词边界匹配
修复后拒保率从 28.2% → 17.5%，排除了 3,045条 误判。
启示：规则系统的真正挑战不在召回而在精度。整词匹配是英文NLP中基础但常被忽视的细节。

风险预测与业务含义

风险量化思路

拒保类 → 高风险 (Risk=1.0)
直接匹配除外标的表 → 不可承保
HS大类 + 历史赔付率加权
例：易碎品(玻璃/陶瓷) → 0.7
机械设备 → 0.3 · 纺织品 → 0.2

业务价值

  • 17.5% 自动筛查为高风险
  • 约14,500条 可自动承保+低风险定价
  • 为保险公司节省约 70% 人工审核时间
局限与改进：当前基于规则的初筛无法捕捉新型风险模式。建议积累标注数据后，引入 XGBoost/LightGBM 构建监督学习风险模型，将分类标签+描述embedding+历史赔付数据作为特征。

总结与展望

已完成

  • 构建了 10类拒保+22类HS 关键词规则库
  • 实现了 83.3% 自动分类覆盖率
  • 发现并修复子串误匹配，提升分类精度

改进方向

  • 引入 TF-IDF 向量相似度兜底未分类
  • 接入 LLM zero-shot 处理高专业度描述
  • 建立人工反馈闭环持续优化词库
核心价值：将非结构化文本 → 结构化风险标签，让机器处理规则、让人专注判断。
